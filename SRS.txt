[SRS v1.1] 구글 닥스용 실시간 인용 관리 솔루션 MVP (업데이트 버전)

1. 개요 (Introduction)
1.1 목적
연구자가 논문 작성 중 문맥 전환 없이 에디터 내에서 참조 문헌을 관리하고, 실시간으로 인용을 삽입 및 검증할 수 있는 구글 닥스 애드온의 요구사항을 정의합니다.

1.2 핵심 가치
'참조 수집 → 집필 및 인용 삽입 → 최종 검증'으로 이어지는 연구 워크플로우를 에디터 내에서 통합하여 연구자의 '몰입 상태(Flow State)'를 유지합니다.

2. 시스템 아키텍처 (System Architecture)
인터페이스: HTML 서비스 기반의 '컨텍스트 유지형' 사이드바 인터페이스를 사용합니다.


데이터 흐름: 사용자가 사이드바에 입력한 텍스트 데이터는 클라이언트 사이드에서 즉시 파싱되어 문서 내 삽입 로직으로 전달됩니다.


3. 기능적 요구사항 (Functional Requirements)
3.1 참조 문헌 풀(Reference Pool) 관리 (신규)
기능: 사용자가 수집한 참조 논문 리스트를 사이드바에 임시 보관(Scrapbook)할 수 있는 영역을 제공합니다.
세부 사항:
자유 서식 복사/붙여넣기: 외부 DB(RISS, Google Scholar 등)에서 복사한 다양한 형식의 문헌 정보를 사이드바 플레이스홀더 영역에 자유롭게 저장할 수 있습니다.
리스트화 및 필터링: 저장된 문헌들을 개별 항목으로 구분하여 시각화하고, 필요한 문헌을 빠르게 찾을 수 있는 검색 기능을 제공합니다.
3.2 원클릭 인용 삽입 (신규)
기능: 사이드바에 모아둔 문헌 중 하나를 선택하여 본문에 즉시 인용구로 삽입합니다.
세부 사항:
자동 스타일 적용: 선택한 문헌 정보를 기반으로 설정된 스타일(예: APA)에 맞는 인용구 (저자, 연도)를 자동 생성하여 현재 커서 위치에 삽입합니다.


더미 인용 연동: 삽입된 인용구는 '기능 3.3(더미 인용 관리)' 시스템에 자동 등록되어, 추후 정식 레퍼런스 리스트 생성 시 추적 대상이 됩니다.
3.3 더미 인용(Dummy Citation) 및 플레이스홀더 관리
기능: 정식 서식이 갖춰지지 않은 임시 인용구(예: Kim:2023)를 감지하고 관리합니다.
세부 사항:
자동 감지: 본문 작성 중 특정 패턴의 임시 인용구가 입력되면 이를 감지하여 사이드바의 '미결 인용 목록'에 추가합니다.
일괄 교체: 집필 완료 후, 사이드바 참조 문헌 풀의 데이터와 매칭하여 문서 전체의 임시 인용구를 정식 인용구로 일괄 전환합니다.
3.4 실시간 인용 자동 대조 (Cross-check)
기능: 본문 내 인용과 문서 하단의 참고문헌 목록 간의 일치 여부를 실시간 검증합니다.


세부 사항: 본문에는 있으나 목록에는 없는 항목, 또는 목록에는 있으나 본문 언급이 없는 항목을 시각적으로 표시합니다.


3.5 다국어(CJK) 특화 처리
기능: 한국어와 중국어 인용 시 발생하는 언어적 특수성을 처리합니다.
세부 사항: 한국어 조사(은/는/이/가 등)를 분리하여 저자명 본체만 추출하고, Unicode NFC 정규화를 통해 기기 간 텍스트 불일치를 방지합니다.


4. 비기능적 요구사항 (Non-functional Requirements)
성능: 사이드바에 적재된 대량의 참조 문헌 데이터가 문서 편집 성능(타이핑 지연 등)에 영향을 주지 않도록 최적화합니다.


사용성: "커피 몇 잔 값으로 밤샘 작업에서 해방된다"는 가치를 체감할 수 있도록 직관적인 UI를 유지합니다.
